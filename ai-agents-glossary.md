## 터미널·셸 계열

### 터미널 (Terminal)
**정의** — 글자를 보여주고 키 입력을 전달하는 창. 명령의 뜻을 해석하지는 않는다.
**비유** 🍽️ — 식당의 주문 창구. 주문을 받아 적어 전달할 뿐, 요리는 안 한다.
**관련 용어**: [[셸]], [[프롬프트(터미널)]]

### 셸 (Shell)
**정의** — 사람이 친 명령을 해석해 프로그램을 실행하는 프로그램의 계열(bash, zsh 등). 커널을 감싼 "껍데기".
**비유** 🍽️ — 웨이터. "김치찌개 주세요"를 알아듣고 주방에 전달한다.
**관련 용어**: [[커널]], [[zsh·bash]], [[터미널]]

### 커널 (Kernel)
**정의** — 운영체제의 핵심. 프로그램이 CPU·메모리·디스크 같은 자원을 쓰도록 중개·통제한다.
**비유** 👨‍🍳 — 주방장(총괄). 요리사(프로그램)들이 화구·재료를 누가 쓸지 배분·통제한다. 직접 요리하기보다 자원을 나눠주는 관리자.
**관련 용어**: [[셸]], [[추상화]]

### 프롬프트(터미널) (Prompt)
**정의** — 셸이 "입력을 기다린다"고 보여주는 표시(`$`, `%`, `#`).
**비유** — "이 위치에서 실행할 명령을 입력하세요"라는 안내. 따라 치는 게 아니다.
**설명** — `%`=zsh, `$`=bash(일반 사용자), `#`=관리자(root, 주의!).
**관련 용어**: [[프롬프트(AI)]], [[sudo]]

### bin·PATH
**정의** — `bin`은 실행 파일(binary)이 담긴 폴더, `PATH`는 명령어를 찾을 때 뒤질 bin 폴더들의 목록.
**비유** 🧰 — bin = 연장통, PATH = 어느 연장통을 열어볼지 정한 목록.
**관련 용어**: [[echo]], [[터미널]]

### pwd
**정의** — Print Working Directory. 지금 있는 폴더의 경로를 출력하는 명령.
**비유** 📍 — "나 지금 어디 있지?" 위치 확인.

### echo
**정의** — 입력한 내용(또는 변수 값)을 그대로 출력하는 명령.
**비유** 🔊 — 메아리. `echo $SHELL`로 환경 변수 값을 확인.
**관련 용어**: [[.env]], [[bin·PATH]]

### sudo
**정의** — Superuser DO. 명령을 관리자(root) 권한으로 실행. 강력한 만큼 신중히.
**비유** 🔑 — 잠긴 방에 들어갈 때만 잠깐 꺼내 쓰는 마스터 키.

### zsh·bash
**정의** — 둘 다 셸. bash는 오래된 표준, zsh는 요즘 Mac 기본(자동완성·꾸미기 강력).
**비유** 🐚 — 같은 일을 하는 통역사, zsh가 더 편의 기능이 많은 신형.
**관련 용어**: [[셸]]

### LTS
**정의** — Long-Term Support. 오래·안정적으로 지원되는 버전.
**비유** 🚗 — 검증되고 정비 지원이 오래 보장되는 자동차 모델.
**관련 용어**: [[Node.js]], [[nvm]]

## 개발 도구·환경

### npm
**정의** — Node Package Manager. JavaScript 패키지(라이브러리)를 설치·관리하는 도구.
**비유** 📦 — 앱스토어에서 앱 설치하듯 코드 부품을 설치.
**관련 용어**: [[Node.js]], [[nvm]], [[JSON]]

### Node.js
**정의** — JavaScript를 브라우저 밖(컴퓨터·서버)에서 실행하는 환경(런타임). 언어가 아니라 실행 환경.
**비유** 🎬 — JavaScript(배우)가 브라우저(극장)뿐 아니라 스튜디오(Node)에서도 연기하게 해줌.
**관련 용어**: [[npm]], [[nvm]], [[Agent]]

### nvm
**정의** — Node Version Manager. Node의 여러 버전을 설치·전환하는 도구.
**비유** 🎮 — 게임 카트리지 갈아 끼우듯 Node 버전을 전환.
**관련 용어**: [[Node.js]], [[LTS]]

### pip·uv
**정의** — Python 패키지 관리 도구. pip은 기본, uv는 Rust로 만든 초고속 신형.
**비유** 🚗 — pip=기본 자동차, uv=최신 전기 스포츠카(훨씬 빠름).
**관련 용어**: [[Python]], [[npm]]

### IDE
**정의** — Integrated Development Environment. 코드 편집·실행·디버깅·Git을 한 프로그램에 통합.
**비유** 🍳 — 싱크대·불·칼·재료가 다 갖춰진 주방. 개발에만 집중.
**관련 용어**: [[LSP]], [[diff]], [[PR]]

### LSP
**정의** — Language Server Protocol. 편집기가 언어를 똑똑하게 이해하도록(자동완성·오류표시) 돕는 표준.
**비유** 🔌 — 언어 이해의 USB. MCP가 여기서 영감을 받음.
**관련 용어**: [[IDE]], [[MCP]]

### diff
**정의** — difference. 두 파일/코드의 달라진 부분(추가 `+`, 삭제 `-`)을 보여주는 것.
**비유** 📝 — 틀린그림찾기. 다른 부분에만 동그라미.
**관련 용어**: [[PR]], [[IDE]]

### 마크다운 (Markdown, .md)
**정의** — `#`, `*`, `-` 같은 기호로 서식을 넣는 쉬운 문서 문법. 최종적으로 HTML로 변환됨.
**비유** 🍜 — 복잡한 요리(HTML) 대신 간단한 라면. 지금 이 답변도 마크다운.
**관련 용어**: [[CSS]], [[PR]]

### .env
**정의** — 비밀번호·API 키 같은 민감정보와 환경 설정값을 담는 파일. 코드와 분리해 보호.
**비유** 🔐 — 집(코드)과 분리된 금고. 절대 Git에 올리지 말 것(.gitignore에 추가).
**관련 용어**: [[API]], [[JWT]], [[.DS_Store]]

### PR (Pull Request)
**정의** — 내 코드 변경을 원본에 합쳐달라는 요청. 검토(리뷰) 후 안전하게 병합.
**비유** ✍️ — 조별 과제에서 "제 부분 검토해 주세요" 하고 제출.
**관련 용어**: [[diff]], [[IDE]]

### .DS_Store
**정의** — macOS가 폴더 보기 설정을 저장하려고 자동 생성하는 숨김 파일. 프로젝트와 무관.
**비유** 📝 — Mac이 남긴 메모지. Git에는 넣지 말고 무시(.gitignore).
**관련 용어**: [[.env]]

## 프로그래밍 언어

### Python
**정의** — 읽기 쉽고 배우기 쉬운 만능 언어. AI·데이터 분석의 사실상 표준.
**비유** 🐍 — 영어 문장처럼 읽히는 코드. 들여쓰기로 블록을 구분.
**관련 용어**: [[pip·uv]], [[객체]]

### JavaScript
**정의** — 웹의 기본 프로그래밍 언어. 브라우저에서 동작을 담당하며, Node.js로 밖에서도 실행.
**비유** 🧠 — 웹페이지의 두뇌·근육(클릭 시 반응 등).
**관련 용어**: [[TypeScript]], [[Node.js]], [[DOM]]

### TypeScript
**정의** — JavaScript에 타입을 더한 언어. 실수를 작성 중에 미리 잡아준다. 실행 전 JS로 변환됨.
**비유** 🛣️ — 가드레일·표지판이 있는 길. 결과물은 결국 JavaScript.
**관련 용어**: [[JavaScript]], [[객체]]

## 언어 개념

### 객체 (Object)
**정의** — 관련된 속성(정보)과 메서드(기능)를 하나로 묶은 것. `{ }` 안에 "키: 값"으로 정의.
**비유** 📦 — 여러 칸으로 나뉜 상자. 속성만·메서드만·빈 객체 모두 가능.
**설명** — 메서드는 "값이 함수인 속성". `dog.name`(속성), `dog.bark()`(메서드).
**관련 용어**: [[const]], [[JSON]], [[해시 테이블]]

### const
**정의** — 변수를 선언하는 키워드. constant(상수) = 재할당 불가. 객체 전용이 아님.
**비유** 🏠 — 주소가 고정된 집. 객체의 내부 속성은 바꿀 수 있어도, 다른 객체로 통째 교체는 불가.
**설명** — const(불변, 기본) / let(가변) / var(옛날).
**관련 용어**: [[객체]]

### 추상화 (Abstraction)
**정의** — 복잡한 내부는 숨기고 필요한 핵심만 보여주는 것.
**비유** 🚗 — 자동차 운전. 핸들·페달만 조작하면 되고 엔진 내부는 몰라도 됨.
**설명** — 컴퓨터는 추상화가 겹겹이 쌓인 구조: 내 코드 → 언어 → 커널 → 기계어.
**관련 용어**: [[커널]], [[API]]

## 웹 기초

### GUI
**정의** — Graphical User Interface. 버튼·아이콘 등 그래픽으로 조작하는 화면 방식(↔ CLI).
**비유** 🖱️ — 앱 아이콘을 눈으로 보고 탭해서 여는 것.

### CSS
**정의** — Cascading Style Sheets. 웹페이지를 꾸미는 언어(색·글꼴·배치).
**비유** 🎨 — 집짓기의 페인트·벽지·가구 배치. HTML(뼈대)에 입힘.
**관련 용어**: [[DOM]], [[GUI]]
*참고: [MDN — CSS](https://developer.mozilla.org/ko/docs/Web/CSS)*

### DOM
**정의** — Document Object Model. HTML을 프로그램이 다룰 수 있게 바꾼 나무(트리) 구조.
**비유** 🎭 — 인형극 무대. HTML(대본)로 세운 인형(DOM)을 JavaScript(조종사)가 움직임.
**관련 용어**: [[CSS]], [[JavaScript]]
*참고: [MDN — DOM](https://developer.mozilla.org/ko/docs/Web/API/Document_Object_Model/Introduction)*

### JSON
**정의** — JavaScript Object Notation. 데이터를 주고받는 글자 형식. 키에 따옴표 필수, 함수 불가.
**비유** 📄 — 객체(가구)를 납작하게 포장한 배송용 설명서.
**관련 용어**: [[객체]], [[API]], [[npm]]

## 네트워크

### LAN
**정의** — Local Area Network. 집·건물 등 좁은 범위의 네트워크.
**비유** 🏠 — 아파트 단지 안 배달(빠름).
**관련 용어**: [[WAN]], [[사설 IP]]

### WAN
**정의** — Wide Area Network. 도시·전세계 등 넓은 범위. 대표적으로 인터넷.
**비유** 🌍 — 도시 간 우편. 작은 LAN들이 WAN으로 연결됨.
**관련 용어**: [[LAN]], [[HTTP]]

### 사설 IP
**정의** — 집 안(로컬)에서만 쓰는 주소(예: 192.168.x.x). 공인 IP 부족을 해결하려고 재사용.
**비유** 🏢 — 회사 내선번호(겹쳐도 됨) ↔ 공인 IP는 대표번호(유일). 공유기가 자동 배정(DHCP).
**관련 용어**: [[LAN]], [[WAN]]

### HTTP
**정의** — HyperText Transfer Protocol. 웹에서 데이터를 주고받는 통신 규약(GET/POST 등).
**비유** 📮 — 정해진 형식의 우편 방식. "어떻게 전달할지"를 정함.
**관련 용어**: [[API]], [[JSON]]

### API
**정의** — Application Programming Interface. 프로그램끼리 소통하는 창구·규칙. "무엇을 요청할 수 있는가".
**비유** 🍽️ — 식당 메뉴판+주문 시스템. 웹 API는 대부분 HTTP를 타고 소통.
**관련 용어**: [[HTTP]], [[JSON]], [[MCP]]
*참고: [MDN — Web API](https://developer.mozilla.org/ko/docs/Web/API)*

## 데이터·보안

### 쿼리 (Query)
**정의** — 데이터(주로 데이터베이스)에 던지는 질문·요청(SQL 등).
**비유** 📚 — 도서관 사서에게 "이런 책 찾아줘" 하는 요청.
**관련 용어**: [[인덱스]], [[PK]]

### 인덱스 (Index)
**정의** — 찾기 쉽게 해주는 색인·위치 번호. 배열 순번(0부터), DB 검색 속도 향상, index.html(첫 페이지) 등.
**비유** 📑 — 책 뒤의 찾아보기.
**관련 용어**: [[쿼리]], [[해시 테이블]], [[PK]]

### PK (Primary Key)
**정의** — 표에서 각 행을 고유하게 구분하는 대표 식별자. 유일·비어있으면 안 됨·안 바뀜.
**비유** 🆔 — 주민등록번호. 짝꿍 FK는 다른 표의 PK를 참조해 연결.
**관련 용어**: [[쿼리]], [[인덱스]], [[RLS]]

### 해시 (Hash)
**정의** — 데이터를 고정 길이의 고유값으로 바꾸는 단방향 변환. 데이터의 "지문".
**비유** 🥤 — 믹서기. 주스(해시)를 과일로 되돌릴 수 없음. 같은 입력=항상 같은 해시.
**관련 용어**: [[해시 테이블]], [[JWT]], [[인덱스]]

### 해시 테이블 (Hash Table)
**정의** — 해시로 저장 위치를 계산해 데이터를 초고속 저장·검색하는 자료구조.
**비유** 🏨 — 호텔 룸키. 이름을 계산하면 방 번호가 바로 나옴. 객체·딕셔너리가 이 원리.
**관련 용어**: [[해시]], [[객체]], [[인덱스]]

### ACID
**정의** — 데이터베이스가 트랜잭션(묶음 작업)을 안전하게 처리하는 4원칙: 원자성·일관성·독립성·지속성.
**비유** 🏦 — 꼼꼼한 은행원의 원칙. 이체는 전부 되거나 전부 취소.
**관련 용어**: [[쿼리]], [[PK]]

### JWT
**정의** — JSON Web Token. 로그인 증명용 디지털 출입증(헤더.페이로드.서명). 내용은 암호화 아님(민감정보 금지).
**비유** 🎫 — 놀이공원 팔찌. 서명으로 위조를 막음.
**관련 용어**: [[세션·토큰]], [[해시]], [[JSON]]

### 세션·토큰
**정의** — 로그인 상태를 유지하는 두 방식. 세션은 서버가 기억, 토큰은 사용자가 정보를 보관.
**비유** — 세션 🗄️ = 클럽 회원 명부(번호표만 소지), 토큰 🎫 = 놀이공원 팔찌(정보 소지).
**설명** — 세션은 통제·강제로그아웃 쉬움, 토큰은 서버 부담 적고 확장 쉬움.
**관련 용어**: [[JWT]], [[쿠키]], [[TTL]]

### 쿠키 (Cookie)
**정의** — 브라우저가 작은 데이터를 저장해두고 요청마다 자동 전송하는 저장 그릇.
**비유** 🍪 — 주머니. 세션 ID(번호표)나 토큰(팔찌)을 담는 그릇. (인증 방식과 차원이 다름!)
**관련 용어**: [[세션·토큰]], [[JWT]]

### TTL (Time To Live)
**정의** — 데이터의 유효기간. 시간이 지나면 자동 만료·삭제.
**비유** 🥛 — 우유 유통기한, 모래시계. 토큰의 "강제 로그아웃 어려움"을 보완.
**관련 용어**: [[JWT]], [[세션·토큰]], [[쿠키]]

### RLS (Row Level Security)
**정의** — 데이터베이스의 행(row) 단위 접근 제어. 각 행에 "누가 볼 수 있는지" 정책을 건다.
**비유** 🔒 — 개인 자물쇠 달린 사물함. Supabase 등에서 필수.
**관련 용어**: [[PK]], [[JWT]], [[.env]]

## AI 계열

### LLM
**정의** — Large Language Model. 방대한 텍스트로 "다음 단어 예측"을 학습해 말을 이해·생성하는 AI 두뇌.
**비유** 📚 — 방대한 책을 읽은 똑똑한 조수. 진실보다 "그럴듯함"을 우선하는 성질.
**관련 용어**: [[RLHF]], [[할루시네이션]], [[Agent]]

### RLHF
**정의** — Reinforcement Learning from Human Feedback. 사람의 평가를 보상 삼아 AI를 친절·유용하게 다듬는 훈련.
**비유** 🐕 — 강아지 훈련. 잘하면 간식(보상)을 줘 그 방향을 강화.
**관련 용어**: [[LLM]], [[벤치마크]]

### 할루시네이션 (Hallucination)
**정의** — AI가 틀린 정보를 사실처럼 그럴듯하게 지어내는 현상.
**비유** 🎭 — 답을 모르는데 자신 있게 지어내는 시험 학생. 중요한 건 반드시 검증.
**관련 용어**: [[LLM]], [[RLHF]]

### 벤치마크 (Benchmark)
**정의** — 여러 AI 모델의 성능을 같은 시험으로 측정·비교하는 표준 시험(MMLU, HumanEval 등).
**비유** 📊 — AI들의 수능. 단, 높은 점수가 실무 성능을 100% 보장하진 않음.
**관련 용어**: [[LLM]]

### 프롬프트(AI) (Prompt)
**정의** — AI에게 "무엇을 할지" 알려주는 지시. 모든 것의 출발점(입력).
**비유** 🧾 — 손님의 주문서. 시스템 프롬프트(역할 설정)+사용자 프롬프트(요청)로 나뉨.
**관련 용어**: [[LLM]], [[Agent]], [[프롬프트(터미널)]]

### Agent (에이전트)
**정의** — 목표를 받으면 스스로 계획·도구사용·반복하며 끝까지 해내는 AI. (일반 LLM은 답만)
**비유** 🧑‍🍳 — 실제로 요리하는 요리사. Claude Code가 대표적.
**관련 용어**: [[LLM]], [[하네스]], [[MCP]]

### 하네스 (Harness)
**정의** — Agent가 실제로 작동하도록 감싸주는 운영 환경·틀(도구 실행, 반복 루프).
**비유** 🐴 — 말(LLM)에 씌우는 안장·마차 연결. 그 힘을 실제 일로 바꿔줌.
**관련 용어**: [[Agent]], [[LLM]], [[스킬]]

### MCP
**정의** — Model Context Protocol. AI와 외부 도구를 잇는 표준 규격(Anthropic).
**비유** 🔌 — AI를 위한 USB 포트. 제각각 → 하나로 통일. LSP에서 영감.
**관련 용어**: [[API]], [[스킬]], [[LSP]]

### 스킬 (Skill)
**정의** — 특정 작업을 잘 하는 방법·노하우를 미리 포장한 것(재사용).
**비유** 📖 — 전문 레시피북. MCP(도구=무엇으로)와 달리 스킬은 방법(어떻게).
**관련 용어**: [[MCP]], [[프롬프트(AI)]], [[Agent]]

### MVP
**정의** — Minimum Viable Product. 핵심 기능만 담아 빠르게 만든 첫 버전.
**비유** 🛴 — 목표가 "이동"이면 킥보드부터 내놓고 반응을 봄.

### CS
**정의** — Computer Science. 컴퓨터·프로그램의 원리를 다루는 학문. "[오늘의 CS]" = 컴퓨터 기초 상식.
**설명** — Customer Service(고객서비스)와 혼동 주의. 개발 맥락이면 Computer Science.
