## 도메인 맥락

나는 서비스/UI기획, 물류 포털 서비스 운영 도메인 20년 차이고, 서비스 기획서(PRD) 초안 작성 작업을 자동화/정리하고 싶습니다.

## 네 동작 분석

1. 분해: 화면, 이슈 유형, 심각도, 처리 상태로 분류
2. 패턴: 반복되는 UI 문제와 원인 도출
3. 추상화: 핵심 개선 포인트 3가지로 요약
4. 알고리즘: 리뷰 → 분류 → 패턴 분석 → 우선순위 → 개선안 출력

## 출력 요구

1. 형식: 한국어로 된 클라이언트 보고서
2. 내용: UI 리뷰 요약, 반복 패턴, 개선 우선순위, 실행 방안
3. 작성 박식: 표와 핵심 문장 중심으로 간결하게 작성

## 검증 기준

서비스 기획 및 UI 운영 전문가 2명이 "보고서 품질이 충분하다"고 평가할 것
