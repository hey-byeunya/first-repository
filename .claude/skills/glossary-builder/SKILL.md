# 용어사전 만들기 (glossary-builder)

## 이 스킬이 하는 일

한 대화에서 개발·CS·AI 용어를 하나씩 설명하다 보면 내용이 흩어진다. 이 스킬은 그렇게 설명된 용어들을 **JSON 데이터**로 쌓아두고, 거기서 **입문자용 한국어 용어사전 HTML**을 생성한다.

핵심은 **JSON이 데이터 저장소(source of truth)**라는 것이다. 용어를 글로 짓는 게 아니라 데이터 항목(단어/정의/비유/관련 용어/참고)으로 등록하고, HTML은 그 데이터에서 매번 다시 만들어진다. 그래서:
- 오늘 대화에서 배운 용어를 **기존 JSON에 추가**하면, 사전이 대화를 넘어 계속 자라난다.
- 이미 있는 용어를 다시 설명했다면 **해당 항목만 갱신**하면 된다 (중복 생성 없음).
- HTML은 손으로 만들지 않는다 — 스크립트가 JSON에서 자동 생성한다.

## 진행 방법

1. **기존 데이터 파일이 있는지 확인한다.** 기본 경로는 프로젝트 루트의 `<이름>.json`이다 (예: `ai-agents-glossary.json`). 있으면 읽어서 이번에 다룰 용어와 겹치는지 확인한다. 없으면 새로 만든다.
2. **용어를 수집한다.** 기본 출처는 **현재 대화**다 — 오늘 설명·논의된 용어를 모은다. 용어가 많거나 범위가 애매하면, 넣을 후보 목록을 먼저 보여주고 확인받는다.
3. **JSON 데이터를 갱신한다.** 아래 "데이터 스키마"에 맞춰:
   - 새 용어면 `terms` 배열에 새 항목을 추가한다.
   - 이미 있는 용어(`term` 값이 같음)면 그 항목을 덮어써서 갱신한다 — 절대 중복 항목을 만들지 않는다.
   - 새 그룹이 필요하면 `groups` 목록에도 추가한다 (순서가 색인·섹션 순서가 된다).
   - 내용 작성 스타일은 [references/style-guide.md](references/style-guide.md)를 따른다 — 첫 사용 전에 반드시 읽는다.
4. **HTML을 생성한다.** JSON을 손으로 수정한 뒤 항상 이 스크립트로 렌더링한다 (손으로 카드를 짜지 않는다):
   ```bash
   python3 .claude/skills/glossary-builder/scripts/json_to_glossary.py <이름>.json <이름>.html --title "탭 제목"
   ```
   - 색인(그룹별 칩), 용어 카드, `related`의 앵커 링크, 표·코드·비유 박스를 자동으로 만든다(라이트/다크 지원).
   - **기본적으로 같은 폴더에 `용어사전.html`이라는 바로가기(심링크)를 만들거나 갱신한다.** 실제 파일명이 뭐든, 이 고정된 이름 하나만 기억하면 항상 최신 사전을 로컬에서 열 수 있다.
5. **로컬에서 바로 보여준다.** 생성 직후 Browser 도구로 `file://<절대경로>/용어사전.html`을 열어 스크린샷으로 확인하고, 채팅에는 클릭 가능한 로컬 링크(`[용어사전.html](용어사전.html)`)를 함께 안내한다.
6. **공유가 필요하면** 생성된 `.html`을 Artifact 도구로 퍼블리시한다. 이미 퍼블리시한 적 있다면 같은 file_path로 다시 퍼블리시해 URL을 유지한다. (로컬 확인만으로 충분하면 이 단계는 생략해도 된다.)

## 데이터 스키마 (JSON)

```json
{
  "title": "개발·CS·AI 용어사전",
  "lede": "이 대화에서 배운 용어들을 비유와 함께 정리했습니다.",
  "groups": ["터미널·셸 계열", "웹 기초", "AI 계열"],
  "terms": [
    {
      "term": "CSS",
      "original": "Cascading Style Sheets",
      "group": "웹 기초",
      "definition": "웹페이지를 꾸미는 언어(색·글꼴·배치).",
      "analogy": {"emoji": "🎨", "text": "집짓기의 페인트·벽지·가구 배치. HTML(뼈대)에 입힘."},
      "details": [
        {"type": "paragraph", "text": "필요하면 추가 설명 문단."},
        {"type": "table", "rows": [["헤더1", "헤더2"], ["a", "b"]]},
        {"type": "code", "text": "h1 { color: blue; }"}
      ],
      "related": ["HTML", "DOM"],
      "references": [{"text": "MDN — CSS", "url": "https://developer.mozilla.org/ko/docs/Web/CSS"}]
    }
  ]
}
```

| 필드 | 필수 | 설명 |
|------|------|------|
| `term` | ✅ | 용어 이름. 색인 칩 텍스트이자 앵커(`#term`)로 쓰인다. |
| `group` | ✅ | 이 용어가 속한 그룹. `groups` 목록의 값과 정확히 일치해야 그 섹션에 묶인다. |
| `definition` | ✅ | 한 줄 정의. |
| `original` | 선택 | 원어·약어 풀이. 제목 옆에 작게 표시. |
| `analogy` | 선택 | `{emoji, text}`. 없으면 비유 박스가 생략된다 — 되도록 채운다. |
| `details` | 선택 | 표(`table`)·코드(`code`)·추가 문단(`paragraph`)의 배열. 순서대로 렌더링. |
| `related` | 선택 | 다른 `term` 이름의 배열. HTML에서 해당 카드로 점프하는 링크가 된다. |
| `references` | 선택 | `{text, url}` 배열. 웹 용어는 MDN 링크를 넣는다. |

인라인 텍스트(`definition`, `analogy.text`, `paragraph`, 표 셀)에는 `**굵게**`, `` `코드` ``, `[텍스트](url)`을 그대로 쓸 수 있다. 다른 용어를 참조할 땐 `related` 배열을 쓰는 게 기본이지만, 본문 문장 중간에서 언급할 땐 `[[용어]]`로도 연결할 수 있다.

## 새 용어 추가·기존 용어 갱신 (자주 하는 작업)

1. `<이름>.json`을 읽는다.
2. `terms` 배열에서 같은 `term` 값을 가진 항목이 있는지 찾는다.
   - 있으면 그 항목의 필드를 갱신(병합)한다.
   - 없으면 새 객체를 추가한다. `group`이 `groups`에 없으면 `groups`에도 추가한다.
3. JSON을 다시 저장한다.
4. `scripts/json_to_glossary.py`로 HTML을 재생성한다 (같은 출력 파일명 유지 → 바로가기가 자동으로 최신을 가리킴).

## 핵심 스타일 규칙 (자세한 건 style-guide 참조)

- **한국어로**, 코딩 입문자가 이해할 수 있게 쉽게 쓴다.
- **비유를 곁들인다** (식당·집짓기·우편·주방 등 일상 비유). 이게 이 사전의 성격이다.
- **웹 기술 용어**(HTML/CSS/JavaScript/DOM/HTTP/API 등)는 [MDN 한국어 문서](https://developer.mozilla.org/ko/docs/Web)를 근거로 삼는다. 필요하면 WebFetch로 확인하고, `references`에 링크를 남긴다.
- **표·짧은 코드·이모지**로 구조화하되, 이모지는 절제해서 쓴다.
- **정확성 우선**: 사실이 헷갈리면 지어내지 말고 확인한다. 특히 약어 풀이·수치·동작 방식.

## 정렬과 색인

- 기본 정렬은 **주제별 묶음**이 읽기 좋다 (예: 터미널·셸 계열 / 웹 3형제 / 데이터베이스·보안 / AI 계열). 묶기 애매하면 **오늘 등장 순서**로 새 그룹을 `groups` 끝에 추가한다.
- 색인·섹션 순서는 JSON의 `groups` 배열 순서를 그대로 따른다 — 순서를 바꾸고 싶으면 이 배열만 바꾸면 된다.

## 기존 마크다운 사전이 있다면 (마이그레이션)

이 스킬이 JSON 기반으로 바뀌기 전에 마크다운으로 쌓아둔 용어사전이 있다면, 한 번만 변환한다:
```bash
python3 .claude/skills/glossary-builder/scripts/migrate_md_to_json.py <이름>.md <이름>.json
```
그 뒤로는 `.md` 파일은 더 갱신하지 않고, JSON을 데이터 저장소로 쓴다.
