AI는 어떻게 스스로 일할까?

두뇌 하나로는 아무것도 못 합니다. 프롬프트 · LLM · Agent · 하네스 · 스킬 · MCP · API가 층층이 맞물려야 비로소 "스스로 일하는 AI"가 됩니다. 식당 하나로 전부 이해해 봅시다.

🍽️ 하나의 비유 — "AI 요리사가 운영하는 식당"

각 개념이 식당의 어느 부분인지 먼저 감을 잡으면, 나머지가 쉽게 붙습니다.

01개념 하나씩

역할 · 한 줄 정의 · 예시 순서로 봅니다.

🧾

프롬프트 (Prompt)

모든 것의 출발점 · 입력

AI에게 "무엇을 할지" 알려주는 지시 = 손님의 주문서. 두뇌(LLM)에 방향과 목표를 넣어주는 입력이라, 프롬프트가 없으면 아무리 좋은 부품도 뭘 할지 모릅니다.

두 종류로 나뉨시스템 프롬프트 🍳 — 하네스가 LLM에게 미리 주는 역할·규칙 설정("너는 친절한 한국어 도우미다"). 사용자 프롬프트 🧾 — 사용자가 그때그때 주는 요청("GUI 뜻 알려줘"). 이 둘이 합쳐져 LLM에 전달됩니다.
🧠

LLM

Large Language Model

방대한 텍스트를 학습해 "다음 단어를 예측"하며 말을 이해·생성하는 AI의 두뇌. 진실보다 "그럴듯함"을 우선하는 성질이 있어요.

특징똑똑하지만 혼자선 갇혀 있음 — 실시간 정보·내 파일·외부 프로그램에 직접 접근 불가. 그래서 나머지 부품이 필요합니다.
🧑‍🍳

Agent

스스로 일하는 AI

목표를 받으면 스스로 계획하고, 도구를 쓰고, 여러 단계를 반복하며 끝까지 해내는 AI. 일반 LLM이 "답만" 한다면 Agent는 "실제로 수행"합니다.

예시 · "여행 계획 짜줘"일반 LLM → 글로 계획을 써주고 끝.
Agent → ① 항공권 검색 → ② 날씨 확인 → ③ 호텔 비교 → ④ 일정표 완성. 스스로 도구를 써가며 여러 단계 수행!
🏪

하네스 (Harness)

운영 환경 · 틀

두뇌(LLM)를 감싸서 실제로 굴러가게 하는 시스템. 도구 실행, 반복 루프, 결과를 받아 다음 행동을 결정하는 구조를 제공합니다.

비유 🐴LLM = 힘센 말(두뇌). 하네스 = 말에 씌우는 안장·고삐·마차 연결. 말만 있으면 뛰기만 하지만, 하네스가 있어야 마차를 끌 수 있습니다.
📖

스킬 (Skill)

방법 · 노하우 (재사용)

특정 작업을 잘 해내는 방법·절차를 미리 포장해둔 것 = 전문 레시피북. 관련 작업이 오면 자동으로 불러와, 매번 처음부터 고민하지 않게 해줍니다.

스킬 vs MCP — 가장 헷갈리는 구분스킬 📖 = "어떻게 하는지" (방법·지식). MCP 🔌 = "무엇으로 하는지" (도구 연결). "된장찌개는 이 순서로 끓여라"(스킬) + "가스레인지를 연결"(MCP) → 둘이 함께 작동! 이 대화의 write-prd·artifact-design도 스킬입니다.
🔌

MCP

Model Context Protocol

AI와 외부 도구를 잇는 표준 연결 규격. Anthropic이 만든 개방형 표준으로, "AI를 위한 USB 포트"라고 생각하면 쉽습니다.

예시슬랙 🔌 · 깃허브 🔌 · 데이터베이스 🔌 — 제각각 만들 필요 없이 같은 MCP 규격으로 연결. 지금 이 대화의 브라우저·디자인 도구도 MCP로 붙어 있어요.
🚪

API

Application Programming Interface

프로그램끼리 정보를 주고받는 창구·규칙. 식당의 주문/납품 창구처럼, 내 프로그램이 외부 서비스와 소통하는 문입니다.

예시 · 날씨 조회"서울 날씨 알려줘" → [날씨 API] → "맑음, 28도" 응답. MCP 서버가 내부적으로 이 API를 호출하기도 합니다.

02가장 헷갈리는 셋 — 프롬프트 · 스킬 · MCP

"AI에게 뭔가를 준다"는 점이 비슷해 보이지만, 주는 게 전혀 다릅니다.

구분무엇을 주나재사용?비유
프롬프트그때그때의 지시 (무엇을 할지)1회성🧾 주문서
스킬작업하는 방법·노하우 (어떻게 할지)재사용 ✅📖 레시피북
MCP외부 도구 연결 (무엇으로 할지)재사용 ✅🔌 도구 콘센트
한 줄 구분법 — 프롬프트는 "무엇을", 스킬은 "어떻게", MCP는 "무엇으로". 예: "PRD 써줘"(프롬프트) → "PRD는 이런 순서로 쓴다"(스킬) → "문서 파일을 생성하는 도구"(MCP).

05실제 서비스로 연결하기

이름 하나가 여러 층에 걸쳐 있다는 점이 핵심입니다.

Claude
LLM하네스

이름이 두 가지를 동시에 가리킵니다 — Anthropic이 만든 두뇌(모델)이자, 그 두뇌를 감싼 대화 앱. claude.ai에서의 대화 = Claude 앱(하네스) 속에서 Claude LLM(두뇌)과 대화하는 것.

Claude Code
Agent

Claude LLM을 두뇌로 삼은 대표적인 Agent — 바로 지금 이 대화 상대! 파일을 읽고, .gitignore를 만들고, 커밋·푸시까지 스스로 수행합니다. 일반 LLM이면 "이렇게 하세요"라고 말만 했겠죠.

ChatGPT
LLM하네스

Claude와 똑같은 구조, 만든 회사만 다름 — 두뇌는 GPT(OpenAI), 그것을 감싼 대화 앱이 ChatGPT. "두뇌 + 이를 감싼 앱"이라는 점이 동일합니다.

노션 (Notion)
API 사용자

자기 LLM을 직접 만들지 않고, 외부 LLM(Claude·GPT 등)을 API로 빌려 씁니다. "요약해줘" 클릭 → 노션이 API로 LLM에 요청 → 결과를 문서에 표시. 수많은 앱이 이렇게 AI 기능을 붙입니다.

분류 팁 🎯 — 앞으로 새로운 AI 서비스를 만나면 두 가지만 물어보세요. ① "LLM을 직접 만들었나, API로 빌려 쓰나?" ② "대화만 하나, 스스로 일하나(Agent)?" 이 두 축으로 대부분의 서비스가 정리됩니다.
