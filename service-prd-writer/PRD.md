# 서비스 기획서 작성기 — 기획서

## 무엇을

서비스 기획자가 문답 형식으로 질문에 답하면, 답변을 모아 서비스 기획서(PRD)를 마크다운 문서로 자동 정리해주는 웹 도구.

## 누가

서비스/기능 기획서를 처음부터 새로 정리해야 하는 기획자. 웹 브라우저만 있으면 설치 없이 바로 사용할 수 있다.

## 질문 흐름

한 번에 한 질문씩, 총 13개 질문에 답한다.

| 순서 | 섹션 | 질문 |
|------|------|------|
| 1 | 기본 정보 | 서비스(기능) 이름 |
| 2 | 배경 | 왜 이걸 만들어야 하는가 (문서에는 "기획 의도"로 표기) |
| 3 | 목표 | 무엇을 이루고 싶은가 |
| 4 | 사용자 | 누가 사용하는가 |
| 5 | 기능 | 핵심 기능은 무엇인가 |
| 6 | 요구항목 | 유저 스토리로 풀면 어떻게 되는가 |
| 7 | 기능 | 사용자 시나리오 (단계별) |
| 8 | 프로세스 | 화면들이 어떤 흐름으로 이어지는가 (문서에는 "주요 프로세스"로 표기) |
| 9 | 화면 | 필요한 화면/구성 요소 |
| 10 | 목표 | 성공 지표(KPI) |
| 11 | 예외 | 예외 상황과 처리 방식 |
| 12 | 범위 | 이번에 제외할 것 |
| 13 | 일정 | 주요 일정/마일스톤 |

각 답변은 형식에 맞춰 자동으로 변환된다: 시나리오는 순서 목록, 요구항목·화면 요소·일정은 목록, 주요 프로세스는 화면을 잇는 박스+화살표 다이어그램, 예외 상황은 "상황 / 동작" 표로 정리된다.

## 화면 요소

### 인트로 화면
- 도구 소개와 사용 안내
- **시작하기** 버튼 — 새 답변으로 시작 (기존 저장 데이터 초기화)
- **이어서 작성하기** 버튼 — 저장된 답변이 있을 때만 노출

### 문답 화면
- 상단 진행률 바와 "N / 13" 카운트
- 현재 질문의 섹션 라벨, 질문, 도움말, 답변 입력창
- **이전 / 다음** 버튼으로 이동, 마지막 질문에서는 **완료하기**로 바뀜
- 입력할 때마다 자동 저장되어 새로고침해도 유지됨

### 결과 화면
- 완성된 기획서를 문서 형태로 미리보기
- **마크다운 복사하기** — 클립보드에 전체 문서 복사
- **.md 파일로 다운로드** — 서비스 이름으로 파일명 생성
- **답변 수정하기** — 문답 화면 처음으로 돌아가 답변 수정 (기존 답변 유지)
- **처음부터 다시** — 확인 후 모든 답변 초기화

## 예외 상황

| 상황 | 동작 |
|------|------|
| 질문에 답하지 않고 다음으로 넘어가면? | 허용된다. 결과 문서에는 "아직 작성되지 않았습니다"로 표시 |
| 브라우저를 닫았다가 다시 열면? | 마지막까지 작성한 답변과 위치가 그대로 유지된다 (localStorage 저장) |
| 결과 화면에서 답변을 수정하고 싶으면? | 답변 수정하기 버튼으로 문답 화면으로 돌아가 이전/다음으로 원하는 질문까지 이동해 수정 |
| 클립보드 복사가 차단된 브라우저 환경이면? | 복사 실패 토스트를 띄우고, 파일 다운로드로 대체 이용 가능 |
| 처음부터 다시를 눌렀는데 실수였다면? | 초기화 전 확인창이 뜨므로 취소할 수 있다 |
