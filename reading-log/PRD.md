# 독서 기록 서비스 — PRD

## 서비스 한 줄 요약

**무엇을**: 읽은 책의 감상과 별점을 기록하고, 나중에 다시 꺼내볼 수 있는 개인 독서 기록 서비스
**누구를 위해**: 책을 꾸준히 읽지만 감상이 메모장·SNS·머릿속에 흩어져 금방 사라지는 독서가
**핵심 화면 3개**: 책 목록 / 책 상세 / 기록 작성·수정

## 기획 의도

지금은 책을 읽고 난 감상을 메모장·SNS·머릿속에 흩어둔 채로 두고 있어, 시간이 지나면 "그 책 어땠더라"를 다시 떠올리기 어렵다. 감상을 남길 창구가 따로 없다 보니 기록 자체를 포기하게 되고, 다음에 비슷한 책을 고를 때도 과거 취향을 참고할 방법이 없다.

이 서비스는 책을 읽자마자 짧게 감상과 별점을 남기고, 나중에 그 기록을 그대로 다시 꺼내볼 수 있는 개인 아카이브를 만들어 이 문제를 해결한다.

## 목표

| 목표 | 지표 | 비고 |
|---|---|---|
| 꾸준한 기록 습관 형성 | 주간·월간 기록 작성 횟수 | 로그인 없는 개인 프로젝트 단계라 로컬 기준으로 측정 |
| 과거 기록 재방문 | 목록 → 상세 화면 이동 비율 | 예전 감상을 실제로 다시 꺼내보는지 확인 |
| (다음 단계) AI 추천 활용 | 추천받은 책을 실제로 목록에 추가하는 비율 | AI 기능이 붙기 전까지는 측정 대상 아님 |

## 개발 범위

**이번에 할 것**
- 책 목록 화면 (상태별 필터: 읽음 / 읽는 중 / 읽고 싶음)
- 책 상세 화면 (감상, 별점, 장르 확인)
- 기록 작성·수정 화면 (새 책 추가, 감상 수정)
- 목업 데이터 기반의 프론트엔드 구현 (실제 DB 연동 없음)

**이번에 안 할 것 (Out of scope)**
- 실제 데이터베이스 연동 — 지금은 목업 데이터만 사용
- 회원가입 / 로그인
- AI 도서 추천, 감상 요약 기능
- 다른 사용자와 기록을 공유하는 커뮤니티 기능
- 책 검색 API 연동 (제목·저자 자동완성 등)

## 요구사항

1. 책을 막 다 읽은 유저가 감상이 휘발되기 전에 남기고 싶어서, 기록 작성 화면에서 감상과 별점을 입력할 수 있어야 한다.
2. 여러 책을 관리하는 유저가 지금 상태(읽음/읽는 중/읽고 싶음)를 한눈에 보고 싶어서, 목록 화면에서 상태별로 필터링할 수 있어야 한다.
3. 예전에 남긴 감상이 궁금한 유저가, 책 상세 화면에서 그날 적어둔 메모 전체를 다시 읽을 수 있어야 한다.
4. 감상을 잘못 적었거나 생각이 바뀐 유저가, 기록 작성·수정 화면에서 기존 기록을 고칠 수 있어야 한다.
5. 다음에 읽을 책을 고민하는 유저가, "읽고 싶음" 상태의 책을 미리 등록해두고 목록에서 나중에 확인할 수 있어야 한다.

```
[책을 다 읽음]
        │
        ▼
┌───────────────────┐
│  기록 작성 화면    │  감상 · 별점 입력
└───────────────────┘
        │ 저장
        ▼
┌───────────────────┐   상태별 필터    ┌───────────────────┐
│   책 목록 화면     │ ─────────────── ▶│  필터링된 목록      │
└───────────────────┘                    └───────────────────┘
        │ 책 카드 클릭
        ▼
┌───────────────────┐   기록 수정 클릭   ┌───────────────────┐
│   책 상세 화면     │ ─────────────── ▶│  기록 작성·수정 화면 │
└───────────────────┘                    └───────────────────┘
```

│ 표지  제목 / 저자   ★★★☆☆    │

```

**책 상세 (와이어프레임)**

┌─────────────────────────────┐

│  감상 입력창                  │
│                              │
│  [      저장      ]           │
└─────────────────────────────┘
```

## 추천 프레임워크

**조건**: 비전공자, 자료·예제 많은 프레임워크 선호, 나중에 백엔드·데이터베이스·AI 기능을 단계적으로 붙일 계획

**추천**: Next.js + Supabase

**왜 이걸 골랐나**
1. 목록·상세 화면을 Next.js로 직접 코딩하면서 실전 감각을 쌓고, 이후 감상 요약·도서 추천 같은 AI 기능을 API Route로 자연스럽게 얹을 수 있는 구조라서 골랐다.
2. React 자체의 학습 곡선은 있지만, 비전공자가 막혔을 때 검색이나 AI 도움으로 뚫고 나가기엔 Next.js만큼 예제가 풍부한 프레임워크가 없어 이 단점이 상쇄된다.
3. DB를 Supabase로 붙이면 서버를 밑바닥부터 만들지 않아도 되어, 배우는 속도와 실제로 완성되는 속도를 동시에 챙길 수 있다.
