Service PRD

수제 디저트 쇼핑몰 — PRD

서비스 한 줄 요약

무엇을

수제 케이크를 중심으로 한 디저트를 판매하는 온라인 쇼핑몰.

누구를 위해

선물용 케이크를 찾는 20~30대.

핵심 화면 3개

  • 상품 목록 — 케이크·쿠키·음료를 훑어보는 화면
  • 상품 상세 — 사진·설명·가격·품절 여부를 확인하는 화면
  • 장바구니 — 담은 상품을 확인하고 수량을 조절하는 화면

기획 의도

지금은 수제 디저트를 SNS DM이나 지인 소개로만 판매하고 있어, 판매 채널이 정식 쇼핑몰 형태로 존재하지 않는다. 그 결과 "여자친구 생일 선물", "부모님 답례 케이크"처럼 선물 목적으로 케이크를 검색하는 잠재 고객에게 전혀 노출되지 않고, 검색해도 뭘 파는지·얼마인지·지금 주문 가능한지 확인할 방법이 없어 결국 눈에 잘 띄는 프랜차이즈 케이크로 발길을 돌리게 된다.

이 서비스는 "선물용 케이크를 검색하다가 우연히 발견해도, 목록만 보고 바로 어떤 디저트가 있는지·구매 가능한지 파악할 수 있는" 온라인 창구를 만들어 이 문제를 해결한다.

목표

목표 지표 비고
선물 검색 유입 확보 "케이크 선물" 등 관련 키워드 검색 노출 수, 월 방문자 수 상품 상세 페이지가 검색엔진에 색인되어야 측정 가능
상품 탐색 → 장바구니 전환 상품 상세 조회 대비 장바구니 담기 비율 이번 단계에서 프론트엔드 이벤트로 측정
(다음 단계) 실제 구매 전환 장바구니 대비 결제 완료 비율 결제 기능이 붙기 전까지는 측정 대상 아님

개발 범위

이번에 할 것

  • 상품 목록 화면 (카테고리 필터: 케이크 / 쿠키 / 음료)
  • 상품 상세 화면 (사진, 설명, 가격, 품절 여부)
  • 장바구니 화면 (수량 조절, 삭제, 합계 금액 계산)
  • 목업 데이터 기반의 프론트엔드 구현 (실제 DB 연동 없음)

이번에 안 할 것 (Out of scope)

  • 실제 결제 연동 (토스페이먼츠 등 PG사 연동)
  • 회원가입 / 로그인
  • 주문 내역 조회, 배송 조회
  • 실제 데이터베이스 연동 — 지금은 목업 데이터만 사용
  • 관리자용 상품 등록 · 재고 관리 화면

요구사항

  1. 선물용 케이크를 찾는 유저가 원하는 상품을 빠르게 훑어보고 싶어서, 상품 목록에서 카테고리(케이크/쿠키/음료)로 필터링할 수 있어야 한다.
  2. 구매를 고민하는 유저가 상품이 품절인지 미리 알고 싶어서, 목록과 상세 화면 모두에서 품절 여부를 바로 확인할 수 있어야 한다.
  3. 선물 받을 사람에게 어울리는지 확인하고 싶은 유저가, 상품 상세 화면에서 사진과 상세 설명·가격을 볼 수 있어야 한다.
  4. 여러 개를 담아 한 번에 사고 싶은 유저가, 장바구니에서 상품 수량을 조절하고 합계 금액을 확인할 수 있어야 한다.
  5. 실수로 담은 상품을 빼고 싶은 유저가, 장바구니에서 상품을 삭제할 수 있어야 한다.
  6. 검색으로 유입된 유저가 원하는 상품 페이지에 바로 도달하고 싶어서, 상품 상세 페이지가 검색 결과에 노출되어야 한다(SEO를 고려한 서버 렌더링 필요).

추천 프레임워크

조건

비전공자 SEO 노출 중요 (선물 검색 유입) 자료·예제 많음 결제·주문 백엔드 확장

추천

Next.js + Supabase

왜 이걸 골랐나

  1. 상품 목록·상세 페이지를 SSR/SSG로 완성된 HTML로 내려줄 수 있어, "선물용 케이크" 같은 검색어로 유입되길 원하는 이 서비스의 핵심 목표에 직접 맞아떨어진다.
  2. React 자체의 학습 곡선은 있지만, 비전공자가 막혔을 때 검색이나 AI 도움으로 뚫고 나가기엔 Next.js만큼 예제가 풍부한 프레임워크가 없어 이 단점이 상쇄된다.
  3. 장바구니까지는 프론트엔드 상태로 처리하고, 이후 결제·주문은 Next.js API Route와 Supabase(주문 테이블)로 자연스럽게 얹을 수 있어 지금 구조를 갈아엎지 않고 확장할 수 있다.
